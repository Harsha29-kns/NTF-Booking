9. System Policies & Enforcement

üö´ One-Time Transfer Policy

graph LR A[Original Owner] -->|First Transfer ‚úÖ| B[Second Owner] B -->|‚ùå Cannot Transfer| C[Third Owner] style A fill:#10b981,color:#fff style B fill:#f59e0b,color:#fff style C fill:#ef4444,color:#fff
Enforcement Mechanism:
  • Smart contract tracks isSecondHand flag
  • Once transferred, flag is set to true
  • Second transfer attempt is blocked on-chain
  • Prevents ticket scalping and resale chains

üë§ One Ticket Per Wallet Policy

graph TD USER[User Wallet] --> CHECK{Has Ticket
for Event?} CHECK -->|No| ALLOW[‚úÖ Can Purchase] CHECK -->|Yes| BLOCK[‚ùå Cannot Purchase] ALLOW --> PURCHASE[Purchase Ticket] PURCHASE --> FLAG[Set hasBoughtTicketForEvent = true] style USER fill:#3b82f6,color:#fff style ALLOW fill:#10b981,color:#fff style BLOCK fill:#ef4444,color:#fff
Enforcement Mechanism:
  • Smart contract maintains hasBoughtTicketForEvent mapping
  • Checked before every purchase
  • Reset only on transfer or refund
  • Prevents bulk buying by single wallet
